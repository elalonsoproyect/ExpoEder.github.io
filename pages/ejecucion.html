The cause-effect graph technique is based on a collection of requirements and is used to determine the minimum possible test cases that can cover a maximum test area of ​​the software.

The main advantage of graphical cause-effect tests is that it reduces test execution time and cost.

This technique aims to reduce the number of test cases, but still covers all necessary test cases with maximum coverage to achieve the desired application quality.
